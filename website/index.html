    Serverless Website Backend
    Website: digitalden.cloud.
    Tech Stack: AWS SAM, Amazon API Gateway, AWS Lambda and Amazon DynamoDB.

    • Used AWS SAM CLI for Infrastructure as Code method & GitHub Actions for CI/CD.
    • The pipeline consists of three jobs; infrastructure testing, build & deploy, and integration testing. Jobs are executed in sequential order and if the tests pass, the SAM application gets packaged and deployed to AWS.
    • The website support a counter of visitors, which displays the latest visitor count. The visitor count value is stored in a DynamoDB database, which is accessed by a Lambda function written in Python3. The function is accessed through a REST API created, which when called will invoke the Lambda function and forward back the direct response due to a “Lambda proxy” configuration.
    • Each time the page is loaded, a short JavaScript script utilizes Fetch API to ping the endpoint of the counter API, before rendering the response in the footer of the page. The website fetches and displays the latest visitor count, while the Lambda function handles incrementation as it interacted exclusively with the database.

    Automating Incident Remediation With AWS Config And AWS Lambda
    Tech Stack: AWS IAM, AWS Config, AWS Lambda, AWS CloudWatch

    • Configured a solution to automatically remediate an incident where a security group's inbound rules no longer conformed with the desired configuration.
    • Used AWS Config and AWS Lambda to monitor and automatically remediate security incidents related to EC2 security groups. Configured AWS Config to monitor changes to specific resources in my AWS account.
    • AWS Config continuously monitors the EC2 security group settings and detects any changes that deviate from the desired configuration. It flags any changes as potential security concerns.
    • Automated the remediation process by integrating AWS Config with AWS Lambda.
    • When AWS Config detected a security incident, such as someone modifying a security group's inbound rules, it triggers the associated Lambda function. The Lambda function then takes remediation actions to bring the security group back into compliance with the desired configuration. For example, revoking unauthorized access.
