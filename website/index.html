    • Frontend to digitalden.cloud. Built on AWS using SAM CLI for Infrastructure as Code and GitHub Actions for CI/CD.
    • Once code is pushed to my Github repo, Github actions are triggered which automates the deployment of files to an S3 bucket and the invalidation of the corresponding CloudFront distribution cache.
    • A simple, functional webpage that hosts multiple links that direct users to external websites. Webpage is written in HTML/JavaScript and is styled in CSS.

    Serverless-Website-Backend
    Website: digitalden.cloud

    • Backend to digitalden.cloud. Built on AWS using SAM CLI for Infrastructure as Code and GitHub Actions for CI/CD.
    • Built on AWS using SAM CLI for IaC method and GitHub Actions for CI/CD. The pipeline consists of three jobs, infrastructure testing, build and deploy and integration testing, which are all executed in sequential order and If the tests pass, the SAM application gets packaged and deployed to AWS.
    • The website has a visitor counter that uses a DynamoDB database and a Lambda function written in Python3 to manage the count. The Lambda function is accessed through a REST API created with API Gateway, which is configured to use a "Lambda proxy" configuration.
    • The website uses a short JavaScript to fetch visitor count from the API endpoint and display it in the footer of the page. This architecture ensures that the visitor count is accurately tracked and avoids any concurrency issues.
